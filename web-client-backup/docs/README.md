## 📝 File Naming Convention

All markdown files follow a consistent naming convention:

- Use lowercase letters
- Separate words with hyphens (`-`)
- Use descriptive names that clearly indicate content

Examples:

- `docker-setup.md`
- `project-structure.md`
- `chat-feature.md`
- `events-feature.md`
